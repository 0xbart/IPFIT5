Pythronic Frequently Asked Questions


Welkom bij de FAQ van Pythronic. Op deze pagina vind u de antwoorden op de meest gestelde vragen.

Algemeen

Door wie Pythronic ontwikkeld?

Pythronic is door vier studenten van de Hogeschool Leiden ontwikkeld, in opdracht van een fictieve opdrachtgever voor project IPFIT5.
  • Bart Mauritz
  • Michael van Huis
  • Wesley Boumans
  • Chester van den Bogaard

Met welk doel is Pythronic ontwikkeld?

Er zijn momenteel een legia forensische tools online beschikbaar. Een veel voorkomend probleem bij deze tools is dat zij niet open-source zijn. Om hier eenheid in te brengen heeft de fictieve opdrachtgever ons benaderd om nieuwe forensische tools te maken waarvan de opdrachtgever de broncode kan inzien.

Is het verkregen bewijs uit de applicatie juridisch bruikbaar?

Pythronic is een triage tool. Dit houd in dat de tool informatie kan verkrijgen uit een informatiesysteem, en geld daarom als een hulpmiddel. Controleer altijd voorafgaand bij gebruik van de applicatie of de juiste bevoegdheden aanwezig zijn om dit hulpmiddel te mogen gebruiken.

Om de authenticiteit van de bestanden te bewaren en achteraf te controleren, is het met Pytronic mogelijk om de unieke waardes (hash waardes) van bestanden te bepalen en weg te schrijven. Bij het vergaren van informatie zal Pytronic enkel gebruik maken van 'read-only' rights. Dit houd in dat het programma niet de bestanden bewerkt en hierdoor de authenticiteit waarborgd.

Gebruikers

Hoe maak ik een nieuwe gebruiker aan?

Een nieuwe gebruiker kan worden aangemaakt middels de volgende stappen:
  • 1. Selecteer '4. Manage users'
  • 2. Selecteer '1. New user'
  • 3. Vul een username in
  • 4. Vul een wachtwoord in

De gebruiker is aangemaakt.

Hoe verander ik mijn username?

Dit is nog niet mogelijk in de huidige versie van Pythronic. Als de username toch veranderd dient te worden, dan is het aan te raden om de gebruiker eerst te verwijderen en vervolgens zelf toe te voegen met de correcte gebruikersnaam.

Hoe verander ik mijn wachtwoord?

In de huidige versie van Pythronic is het niet mogelijk om het wachtwoord middels de interface te wijzigen; dit is enkel mogelijk door deze te wijzigen in de database.

Hoe verwijder ik een gebruiker?

Een bestaande gebruiker kan worden verwijderd middels de volgende stappen:
  • 1. Selecteer '4. Manage users'
  • 2. Selecteer '2. Delete user'
  • 3. Selecteer 'P' om de gebruiker permanent te verwijderen, of 'Y' om de gebruiker normaal te verwijderen.
    • De gebruiker is verwijderd.

Als ik een gebruiker verwijder, zijn zijn cases dan nog toegankelijk?

Nee, de informatie van de cases staan opgeslagen in de database.
Case

Wat is een case?

Een case is een digitale ruimte waar informatie over je zaak wordt opgeslagen. Denk hierbij aan informatie welke is verkregen uit een informatiesysteem.

Hoe kan een nieuwe case worden aangemaakt?

Een case kan door middel van de volgende stappen worden aangemaakt:
  • 1. Selecteer 1. New case
  • 2. Vul een case naam in
  • 3. Vul een description in
De case is nu aangemaakt.

Hoe kan een bestaande case worden verwijderd?

Een case kan door middel van de volgende stappen worden verwijderd:
  • 1. Selecteer 3. Delete case
  • 2. Selecteer de case
  • 3. Maak de keuze of de case permanent wordt verwijderd (P) of normaal wordt verwijderd (Y)
De case is nu verwijderd.

Waar staan de gegevens opgeslagen?

De gegevens van de users en cases worden opgeslagen in database bestand pythronic.db, welke te vinden is in de map db/. De verkregen bewijslast wordt opgeslagen in een eigen database bestand, welke te vinden is in de map db/cases/.

Help! Ik heb een case verwijderd terwijl dat niet de bedoeling was. Ben ik dit nu kwijt?

Maak je geen zorgen. Wij hebben er voor gezorgd dat de case enkel uit het keuze overzicht is verdwenen, maar je gegevens staan nog wel bewaard in de database.
Door de database door middel van een SQLite browser te openen, zijn de gegevens te vinden. Door bij de case een 0 in een 1 te veranderen, verschijnen de gegevens weer in de applicatie.

Voor wie is mijn case toegankelijk?

Momenteel is een case voor alle gebruikers toegankelijk. Wij zullen dit in de toekomst verhelpen.

Kan ik meerdere gebruikers toegang geven tot mijn case?

Momenteel is een case voor alle gebruikers toegankelijk, waardoor dit al mogelijk is.
Modules

Clipboard

De clipboard functionaliteit schrijft de aanwezige informatie in de clipboard weg naar bestand clipboard.txt.

Clipboard werkt op: Windows, OSX, Linux

Drive detect

Drive detect analyseert het systeem op aanwezige schrijven, en laat deze zien aan de onderzoeker.

Werkt op: Windows, OSX, Linux

Exif data

Afbeeldingen bevatten vaak meer informatie dan dat het oog laat zien. Het is bijvoorbeeld mogelijk dat deze GPS coordinaten bevat. Deze staat opgeslagen in de zogenaamde EXIF data.
De module EXIF data is er voor geschreven om deze GPS coordinaten van de afbeeldingen te indexeren en in een overzicht weer te geven, waardoor het voor de onderzoeker mogelijk is om 1 oog opslag te zien waar de foto's zijn geschoten.

Werkt op: Windows, OSX, Linux

File hashing

File Hashing stelt de gebruiker in staat op unieke waardes, zogenaamde hashes, te onttrekken vanuit de code en vervolgens op te slaan. Deze codes kunnen worden gebruikt om de integriteit van een bestand te controleren.

Werkt op: Windows, OSX, Linux

Internet history

De module ‘Internet history’ geeft de onderzoeker inzicht in, indien aanwezig, de volgende webbrowsers:

  • Internet Explorer
  • Firefox
  • Google Chrome
  • Safari


Werkt op: Windows, OSX, Linux

Start-up programs

Wanneer een computer wordt opgestart, worden er ook een aantal applicaties opgestart. Welke applicaties dit zijn, kan worden achterhaalt met de ‘Start-up’ module.

Werkt op: Windows

Network connection

Om de onderzoeker inzicht te geven in de netwerkverbindingen van het systeem, kan ‘Network connections’ worden aangeroepen. Deze module toont de IP adressen welke actief zijn op het systeem.

Werkt op: Windows, OSX, Linux

Time-zone

Om de tijd van bewijsstukken te kunnen interpreteren, is het van belang dat de systeemtijd bekend is. De module timezone brengt de exacte tijd en datum van de computer in kaart.

Werkt op: Windows, OSX, Linux

System information

System information is geschreven om de onderzoeker inzicht te geven in de hardware van het te onderzoeken systeem. Er wordt informatie van de volgende onderdelen opgevraagd;
  • PC name
  • Username
  • System architecture
  • Operating system
  • Processor name
  • Processor family
  • Total virtual memory in Gb
  • Used virtual memory in Gb
  • Available virtual memory in Gb


Werkt op: Windows

Usernames

Een computer kan meerdere gebruikers bevatten. De module Usernames brengt deze voor de onderzoeker in kaart.

Werkt op: Windows, OSX, Linux

VirusTotal

Virus controle stelt de onderzoeker in staat om te controleren of een bestand is besmet met een virus of andere kwaadaardige software. De onderzoeker kan een bestand selecteren, waarna de data automatisch naar Virustotal.com wordt geüpload en wordt geanalyseerd.

Let op: Er dient een internet verbinding aanwezig te zijn

Werkt op: Windows, OSX, Linux

Cloud services

Om onderzoek te doen naar welke clouddiensten er op een computer draaien, is de Cloud controle toegevoegd. Deze module stelt te gebruiker in staat om te controleren of de volgende clouddiensten actief zijn op de computer:
  • OneDrive
  • Evernote
  • Dropbox
  • Google Drive

Cloud controle werkt op: Windows, OSX, Linux

Mousejiggler

Tijdens een onderzoek dient te worden voorkomen dat een computer in slaapstand raakt, omdat dit een beveiliging zou kunnen activeren. Om dit te voorkomen, is de mousejiggler toegevoegd.

De mousejiggler zorgt ervoor dat de muis continue beweegt zodat de computer denkt dat er een gebruiker achter de computer zit. Aan de hand van een vooraf ingestelde aantal pixels beweegt de muis.

Werkt op: Windows, OSX, Linux
